# MANIFEST – tof-legacy-recovery-workbench

## Purpose

This workbench recovers technically usable substance from older sources, separates mixed legacy layers, and prepares traceable outputs for later planning or review.

## Required chain

1. universal intake
2. neutral evidence
3. open-set hypotheses
4. split extraction
5. mapping to stable target classes
6. review / acceptance
7. optional later downstream materialization outside the default run

## Stable target classes

- discord
- bot
- repo
- review_required

## Not allowed

- blindly mirror old structure
- ignore files without an extension
- treat unknown patterns as active target families
- count generic stub code as success
- silently turn recovered material into runtime truth

## Data style

Append-only JSON artifacts per stage. No SQLite, no local DB fallbacks.
